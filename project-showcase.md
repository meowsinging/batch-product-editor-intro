# 批次上架編輯器——一套工具，同時搞定四個平台

> 適合閱讀對象：不懂程式碼，但想知道這套系統能做什麼的人。

---

## 你以為上架商品很簡單？

每次上架一款商品，你需要：

- 登入平台1後台，一個欄位一個欄位填進去
- 再登入 平台2後台，重複一次
- 再登入 平台3後台，重複一次
- 再登入 平台4 後台，重複一次
- 上傳圖片到四個地方
- 平台1的定價有一套特殊規則要套用（其他平台目前直接沿用原價）
- 確認沒有欄位填錯

很多人的做法是複製既有的賣場再修改，但這樣更危險——**舊欄位沒改到、舊圖片沒換掉、舊描述留著**，上架了才發現賣錯商品。

**一款商品，四個後台，重複做四次。手動上架上 50 款要花多少時間？**

這套系統的任務，就是把這些事情壓縮成「按一個鍵」。

---

## 亮點一：「一鍵四平台」——四份資料同時產出，不用多按第二下

過去要上架一款商品，得先登入平台1後台，一個欄位一個欄位貼上去；貼完再去平台2 / 平台3後台各重複一次；再去平台4重複一次。欄位名稱不同、格式不同、連定價規則都不同。  
現在：匯入商品資料 → 按下匯出 → 平台1、平台2、平台3、平台4 四份檔案同時下載完畢。

每個平台的規則全部寫死在系統裡：

- **平台1**：產出官方規定的 xlsx 模板格式，112 個欄位全部自動填好
- **平台2 / 平台3**：各有一份，欄位格式、圖片處理、分類都不同，各自獨立產出
- **平台4**：Excel 匯入檔 + 圖片 ZIP 壓縮檔，一次打包下載

**你不需要記任何平台規則，系統全部幫你記好了。**

---

## 亮點二：「平台1定價機器人」——不是亂加價，是有邏輯的漲

平台1的定價不能跟其他平台一樣，有一套調整邏輯：

- 售價 < 1000 → 乘以 1.1
- 售價 1000–5000 → 乘以 1.05
- 售價 5000–10000 → 加 300
- 售價 > 10000 → 加 600
- 最後還要無條件進位到「0、5、9」結尾的數字

舉個例子：一款原價 850 的商品，850 × 1.1 = 935，再進位到 939。  
這種計算如果用人工做，100 款商品就要算 100 次，算錯一個還不一定發現。

這套邏輯寫在程式裡，**每一筆商品都自動套用，不會算錯，也不會忘記**。未來如果規則要調整，改一個地方，全部商品立刻跟著更新。

---

## 亮點三：「勾選就能合體」——多款單規格商品，一鍵變成一個多規格商品

實際情境：同一款模型有「A版」「B版」「C版」，原本是三筆獨立商品。  
但你希望合成一筆，讓買家在同一個商品頁面裡選版本，管理起來也更乾淨——四個平台都支援這種做法。

以前的做法：手動把三筆資料複製整合，規格名稱、SKU、價格、庫存要一格一格搬，容易出錯又費時。  
現在的做法：**在清單上打勾 → 點「合併為多規格」→ 完成**。

系統自動處理：
- 三筆商品的規格選項自動對應（A版、B版、C版變成下拉選項）
- 每個規格的 SKU、售價、庫存各自保留
- 匯出時四個平台同步以多規格格式輸出，不用再手動調整

---

## 亮點四：「批次改分類」——100 個商品，一秒全部改好

每個平台的分類樹都不一樣，而且一款商品要同時維護四個分類（平台1、平台2、平台3、平台4）。

實際情境：新到一批玩具模型，全部都要掛到同一個子分類下。  
以前：一筆一筆點進去改，改完平台1再改平台2 / 平台3，再改平台4，重複 N 次。  
現在：**把其中一筆的四個平台分類設好 → 全部打勾 → 按「同步商店分類」→ 全部同步完畢**。

系統以第一筆勾選的商品分類為基準，一次套用到所有勾選商品，四個平台同步，**不管幾百筆都是一秒的事**。

---

## 亮點五：「圖片自動對號入座」——上傳一次，四個平台各取所需，格式全部處理好

圖片是最麻煩的事。每個平台要的格式完全不同，處理起來像在走迷宮。

這套系統支援每款商品最多 **20 張圖片**，上傳後自動處理以下事情：

- **統一規格**：每張圖片自動調整為 **750px 寬**，尺寸不足的補上白邊，格式統一轉成 **JPG**，不用事前用 Photoshop 一張一張改
- **平台1 / 平台2 / 平台3**：需要的是圖片網址，系統透過 FTP 傳到圖床後，自動建立「商品編號 → 圖片網址」的對應表，匯出時自動填入正確欄位
- **平台4**：需要的是圖片壓縮檔，系統自動把圖片打包成 ZIP 檔案一起下載，平台直接吃

**上傳一次，四個平台各拿各需要的格式，不用複製，不會貼錯。**

---

## 亮點六：「你不可能手寫的格式，系統幫你寫好」——HTML 編輯器 + 免費翻譯一條龍

各平台的商品說明欄位，底層要求的是 **HTML 語法**——也就是一般人根本看不懂、更不可能手打的程式碼格式。  
如果要你直接填，大概會長這樣：

```
<p style="text-align:center;"><span style="font-family:微軟正黑體;">商品說明</span></p>
```

這套系統整合了一個開源的富文字編輯器（Rich Text Editor），**讓你像打 Word 一樣編輯內容，系統自動在背後轉成正確的 HTML 格式輸出**。

你可以做到：
- 調整字體大小、顏色、粗體、對齊
- 插入商品圖片到說明文內
- 不同平台套用不同的品牌色（例如平台3用綠色主題，平台2用預設色）
- 輸出時自動將顏色統一成該平台的設定，不會貼錯顏色

另外，如果商品說明需要日文或簡體中文版本，系統可以直接呼叫翻譯 API 自動產出——**而且完全不花錢，用的是免費額度**。

---

## 亮點七：「官方改格式？十分鐘內跟上」——不是靠運氣，是靠架構

電商平台的官方匯入格式會不定期異動。以這次為例，平台1官方移除了「交通工具類型」欄位，從 113 欄縮成 112 欄。

如果系統設計不好，這種改動可能要花好幾個小時找哪裡壞掉。  
因為這套系統的架構設計清楚——每個平台的欄位定義、模板檔案、資料填入邏輯各自獨立——這次的修改只需要：

- 改一行程式碼（欄位清單）
- 換一個 Excel 模板檔案
- 上傳，全部更新完畢

**沒有大工程，沒有副作用，因為每個部分各司其職，動一個不影響其他。**

這也意味著：未來平台再改格式，維護成本永遠可控。

---

## 亮點八：「Docker 啟動即更新」——打開電腦就是最新版，不需要人工更新

這套系統跑在一個叫做 Docker 的容器裡。簡單說，Docker 就像一個「隨身帶著完整環境的盒子」，不管在哪台電腦上打開，跑起來都一模一樣。

更重要的是：**每次啟動，它會自動去抓最新的程式版本**。

- 修了一個 bug？隔天開機自動修好
- 加了新功能？隔天開機自動有
- 不需要手動下載、不需要安裝、不需要問「我要去哪裡更新？」

對使用者來說，**永遠只有一個動作：打開 Docker，然後開始用**。

---

## 亮點九：「測試不影響正式」——新功能在平行宇宙裡先跑

每次要開發或測試新功能，系統會開一條獨立的「測試軌道」（feature 分支）。  
這條軌道跟你平常用的正式版本完全隔離，在上面怎麼改、怎麼壞都不影響你的日常作業。

測試沒問題了，才正式合併進主版本，Docker 啟動時自動拿到更新。

**這就是為什麼你幾乎不會看到「昨天還好好的，今天突然壞掉」的狀況**——因為每一個新功能都先在隔離環境驗證過，沒問題才讓你用到。
